# QuantOS CN 与同类产品对比

本文帮助你在选型时理解 QuantOS CN 的定位与优势。对比基于公开产品形态与常见开源量化栈（2025–2026），非商业贬损。

---

## 对比总表

| 维度 | **QuantOS CN** | JoinQuant / RiceQuant 等云端 | vn.py + 自写脚本 | MiniQMT / PTrade 券商终端 |
|------|------------------|-------------------------------|------------------|---------------------------|
| **部署** | 本地优先 | 云端 SaaS | 本地 | 券商分发 |
| **A 股规则** | T+1、涨跌停、一手内置 | 平台封装 | 需自实现 | 平台封装 |
| **小资金选股** | 按资金算一手、任意价区 | 有筛选但逻辑不透明 | 需自写 | 偏手动 |
| **研究→验证→辅助实盘** | Paper→Shadow→票据→对账 | 仿真→实盘（视平台） | 无标准链 | 直接交易 |
| **持续学习** | preset 次日收益账本 | 部分有因子库 | 无 | 无 |
| **数据门禁** | stale 拦截交易路径 | 平台保障 | 无 | 无 |
| **多智能体研究** | 9 角色 + 文献框架 | 少见 | 无 | 无 |
| **订单审计 / 对账** | 票据 + CSV 对账 | 平台订单记录 | 需自写 | 券商记录 |
| **自动真实下单** | **禁用（人工确认）** | 可开通 | 可实现 | 可实现 |
| **数据隐私** | 本机 DuckDB | 上传策略/数据 | 本机 | 本机 |

## 我们的核心优势（重点）

### 1. 不是「只能回测」的研究玩具

- **真实 DuckDB 日线仓库** + 多因子选股（非硬编码股票池）
- **网关回测** 接入 screener 组合逻辑（Sharpe、回撤、涨停 blocked 统计）
- **模型实验室** 样本外 + 滚动 + Paper/Shadow 样本门控

对标 Alpha-Forge / SAM 的 **research → backtest_passed → shadow → paper** 推广链，但针对 **A 股规则** 落地。

### 2. 不是「只能云端」的黑盒

- 策略逻辑、学习账本、Paper 状态在 **`data/gateway/`**（已 gitignore，不上传）
- 可选 Tushare / AKShare，不绑定单一商业数据商
- 适合：**不想把因子和持仓交给第三方云平台的个人研究者**

### 3. 不是「一键全自动炒股」的违规承诺

- **订单票据 + 券商官方确认** — 与 Inalpha 等「LLM 不直连发单」原则一致
- **成交导入对账** — 辅助实盘可审计，而非 silent auto-trade
- 明确 **REAL_MONEY_EXECUTION_DISABLED**

### 4. 小资金友好（差异化体验）

- 5000 元也能在 **3–10 元** 等任意区间做量化排序
- 表格展示 **建议一手、一手成本、因子贡献** — 透明可解释

### 5. 持续学习，而非一次性报告

- 每次选股写入 **`learning_screener_runs.jsonl`**
- 自动对比 `balanced / momentum / trend / low_vol` 的次日表现
- 总览给出 **切换 preset / 收紧价区** 等可执行建议

### 6. 一体化 Web 门户（操作简单）

- 单端口 **8787**：总览、选股、Paper、券商、指南
- **一键新手流程**：体检 → 数据 → 选股 → Paper → 票据
- 无需改代码即可完成日常操作

## 适用人群

| 更适合 QuantOS CN | 可能更适合其他方案 |
|-------------------|---------------------|
| 个人 A 股量化研究、重视隐私 | 机构高频、需托管 colocation |
| 小资金、要透明因子逻辑 | 只要云端一键实盘、接受黑盒 |
| 希望 Paper 验证后再人工下单 | 需要完全无人值守自动交易 |
| 愿意本地维护数据与环境 | 不想装 Python / DuckDB |

---

## 与开源项目的关系

- **vn.py / Qlib**：QuantOS 提供适配与门户编排，非替代
- **agentic_quant_platform / SAM**：借鉴推广链与 MLOps 思想；QuantOS 专注 **A 股 + 人工确认辅助实盘**
- **Inalpha**：借鉴审计与 harness 思路；QuantOS 用规则化多智能体 + 真实 screener 输出

---

## 总结一句话

**QuantOS CN = 本地 A 股量化工作台 + 可学习 + 可验证 + 可对接券商的人工确认闭环** — 把「纸上谈兵」推进到「可监控、可对账的辅助实盘」，而不是推销全自动炒股。
